Your Process — GitHub AI-SDLC on the Microsoft Stack

The executive summary: how structured AI-assisted development works with the tools you already have.

Both Approaches, Same Goal

AWS AI-DLC and GitHub AI-SDLC solve the same problem — moving from unstructured AI chat to a repeatable workflow. They're complementary, not competing.

AWS AI-DLC

Your Process: Research → Plan → Implement (RPI)

The core takeaway. Three phases, each with a single clear goal and focused context.

🔍
Research
Full codebase access. Agent explores architecture, maps dependencies, produces research.md. Use @workspace, /explain, LSP queries.
→
📐
Plan
Read-only research.md input. Agent drafts implementation plan with ordered steps, risk register, verification commands. Produces plan.md.
→
⚡
Implement
Follows plan.md step-by-step. Agent mode or coding agent. Harnesses (compiler + tests + linters) run after every change. Self-corrects from errors.

The key insight: Each phase gets a fresh context with only the artefacts from the previous phase. This is an "information diet" — the agent sees only what it needs, reducing hallucination and drift.
